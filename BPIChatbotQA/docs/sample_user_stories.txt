## User Story: US-001 — User Registration

**As a** new user,
**I want to** register an account with my email and password,
**So that** I can log in and access the BPI system.

### Acceptance Criteria:
1. Email must be a valid format (user@domain.com).
2. Password must be minimum 8 characters, contain at least one uppercase letter, one number, and one special character.
3. If the email already exists, the system shall display the error: "An account with this email already exists."
4. On successful registration, the user receives a confirmation email within 2 minutes.
5. The user account is inactive until the email is confirmed.

---

## User Story: US-002 — Login

**As a** registered user,
**I want to** log in using my credentials,
**So that** I can access my dashboard.

### Acceptance Criteria:
1. On 3 consecutive failed login attempts, the account is locked for 15 minutes.
2. The locked account shows the message: "Your account has been temporarily locked due to multiple failed attempts. Try again in 15 minutes."
3. Password is case-sensitive.
4. Session expires after 30 minutes of inactivity.
5. On session expiry, the user is redirected to the login page with the message: "Your session has expired. Please log in again."

---

## User Story: US-003 — Payment Processing

**As a** user,
**I want to** make a payment for my order,
**So that** my order is confirmed.

### Acceptance Criteria:
1. Supported payment methods: Credit Card, Debit Card, Bank Transfer.
2. If the payment gateway returns an error, the system displays: "Payment could not be processed. Please try again or use a different payment method."
3. A payment confirmation is emailed to the user within 5 minutes of a successful transaction.
4. Duplicate payments (same order within 10 minutes) are blocked and the user is shown: "This order has already been paid."
5. Partial payments are not allowed — the full amount must be paid in one transaction.

---

## User Story: US-004 — Order Submission

**As a** user,
**I want to** submit an order,
**So that** it is processed by the system.

### Acceptance Criteria:
1. All mandatory fields (product, quantity, delivery address) must be filled before submission.
2. Quantity must be a positive integer (1 to 999).
3. The system validates stock availability before confirming the order.
4. If stock is insufficient, the system shows: "Requested quantity exceeds available stock. Available: X units."
5. On successful submission, an order number is generated in the format: ORD-YYYYMMDD-XXXXX.

## System Design: Error Handling

- All validation errors are displayed inline next to the relevant field.
- System errors (HTTP 500) display a generic message: "An unexpected error occurred. Please contact support."
- All errors are logged with a correlation ID for support tracing.
- The API returns standard HTTP status codes: 200 (success), 400 (validation error), 401 (unauthorized), 403 (forbidden), 404 (not found), 500 (server error).

## System Design: Integration with External Systems

- **Payment Gateway**: Integrates via REST API. Timeout: 30 seconds. On timeout, the transaction is marked "Pending" and reconciled within 1 hour.
- **Email Service**: SendGrid. Retries up to 3 times on failure with 5-minute intervals.
- **Inventory System**: Synchronizes every 5 minutes. Real-time stock check at order submission.
